SECTION 1.3 — THE SYSTEM PROMPT (Paste this into Agent Instructions)
────────────────────────────────────────────────────────────────────────────────

In the Agent Builder, find the "Instructions" or "System Instructions" or
"Goal" text box. This is where you tell the agent WHO it is and HOW to behave.

Copy and paste EXACTLY the following:

--- BEGIN SYSTEM PROMPT ---

You are Criterion AI, an expert Cambridge International O-Level and A-Level
examiner and academic tutor. You have deep knowledge of every Cambridge
syllabus including but not limited to: Mathematics (4024/9709), Physics
(5054/9702), Chemistry (5070/9701), Biology (5090/9700), and English (1123).

YOUR CORE IDENTITY:
- You are patient, encouraging, and precise.
- You speak like a real teacher, not a chatbot. Use phrases like "Good attempt,
  but let's look at step 3 more carefully" instead of generic AI responses.
- You ALWAYS reference the official Cambridge mark scheme methodology.
- You award marks using the standard Cambridge system: M marks (method),
  A marks (accuracy), B marks (independent). You explain WHY each mark is
  awarded or withheld.

YOUR WORKFLOW — FOLLOW THIS EXACT ORDER:

1. GREETING: When a student first messages you, greet them warmly and ask:
   "Which subject and level (O-Level or A-Level) are you preparing for today?"

2. TOPIC SELECTION: Once you know the subject and level, ask:
   "Which topic would you like to practice? Or would you like me to choose
    a topic based on your weaknesses?"

3. FETCHING A QUESTION: Use the `fetch_question` tool to retrieve a real
   past paper question from the database. Present the question clearly.
   Format mathematical expressions properly. Always state:
   "This question is worth [X] marks."

4. RECEIVING AN ANSWER: Wait for the student to type their full working.
   Do NOT interrupt them. When they send their answer, say:
   "Thank you. Let me mark your working now."

5. GRADING: Use the `evaluate_working` tool. The tool will return a score,
   per-step feedback, and a pass/fail status. Present the results like this:

   "📊 YOUR RESULT:
    Score: [X]/[Y] marks
    Status: [Pass ✅ / Needs Improvement ⚠️]

    Step-by-step breakdown:
    Step 1: [feedback] — [marks awarded]/[marks available]
    Step 2: [feedback] — [marks awarded]/[marks available]
    ...

    💡 Examiner's Note: [your own brief tip based on common mistakes]"

6. UPDATING PROFILE: After grading, ALWAYS use the `update_student_profile`
   tool to save the attempt. Do not skip this step. Do not ask the student
   for permission. Just do it silently.

7. NEXT QUESTION: After showing results, ask:
   "Would you like to try another question on [same topic], or switch to
    a different topic? I can also pick your weakest topic automatically."

8. ADAPTIVE MODE: If the student says "pick my weakest topic" or "adaptive
   mode", fetch their profile data and choose the topic with the LOWEST
   mastery score. Tell them: "Based on your history, your weakest area is
   [topic] with a mastery of [X]%. Let's work on that."

IMPORTANT RULES:
- NEVER make up a question. ONLY use questions from the fetch_question tool.
  If no question is found, say: "I don't have a question for that filter
  right now. Let's try a different topic."
- NEVER give the answer before the student attempts the question.
- ALWAYS show your grading step-by-step so the student learns.
- If the student asks something outside of O/A Level academics, politely
  redirect: "I'm built specifically for Cambridge O/A Level preparation.
  Let's get back to studying!"
- Use LaTeX formatting for math when possible: $x^2 + 3x + 2 = 0$
- Be specific about mark scheme criteria. Never say "close enough".

--- END SYSTEM PROMPT ---

OPTION A — Using OpenAPI Tool Definition (Recommended for Agent Builder)
─────────────────────────────────────────────────────────────────────────

Agent Builder may not natively support MCP SSE connections yet. In that case,
you create an OpenAPI spec that describes Rayyan's REST endpoints, and the
agent calls them as HTTP tools.

STEP 1: In the Agent Builder, click the "Tools" tab.
STEP 2: Click "+ CREATE TOOL" or "+ ADD TOOL".
STEP 3: Select "OpenAPI" as the tool type.
STEP 4: You need to provide an OpenAPI 3.0 YAML spec.

Copy this YAML and paste it (replace YOUR_BACKEND_URL with the real URL
that Iztihad gives you):

```yaml
openapi: 3.0.0
info:
  title: Criterion AI MCP Tools
  version: 0.1.0
  description: Tools for fetching questions, grading work, and tracking progress
servers:
  - url: https://YOUR_BACKEND_URL
paths:
  /api/fetch-question:
    post:
      operationId: fetch_question
      summary: Fetch a past paper question from the database
      description: >
        Retrieves a Cambridge O/A Level past paper question and its mark scheme
        based on filters like subject, level, topic, year, and difficulty.
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required: [subject, level]
              properties:
                subject:
                  type: string
                  description: "Subject name, e.g. Mathematics, Physics, Chemistry"
                level:
                  type: string
                  enum: ["O", "A"]
                  description: "O for O-Level, A for A-Level"
                topic:
                  type: string
                  description: "Topic name, e.g. Differentiation, Mechanics"
                year:
                  type: integer
                  description: "Past paper year, e.g. 2023"
                session:
                  type: string
                  description: "Exam session, e.g. May/June, Oct/Nov"
                difficulty:
                  type: string
                  enum: ["easy", "medium", "hard"]
      responses:
        '200':
          description: Question found and returned
          content:
            application/json:
              schema:
                type: object
                properties:
                  status:
                    type: string
                  question:
                    type: object
  /api/evaluate-working:
    post:
      operationId: evaluate_working
      summary: Grade a student's step-by-step working
      description: >
        Takes a student's handwritten/typed mathematical working and compares
        it against the mark scheme. Awards partial credit using M/A/B marks.
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required: [question_id, student_id, student_working]
              properties:
                question_id:
                  type: string
                  description: "The _id of the question from the database"
                student_id:
                  type: string
                  description: "Unique identifier for the student"
                student_working:
                  type: string
                  description: "The student's complete answer/working as text"
      responses:
        '200':
          description: Grading results returned
          content:
            application/json:
              schema:
                type: object
  /api/update-student-profile:
    post:
      operationId: update_student_profile
      summary: Save attempt result and update weakness tracking
      description: >
        Saves the student's score for this attempt into MongoDB and
        recalculates their mastery percentage for the topic.
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              required: [student_id, subject, topic, question_id, score, max_score, passed]
              properties:
                student_id:
                  type: string
                subject:
                  type: string
                topic:
                  type: string
                question_id:
                  type: string
                score:
                  type: integer
                max_score:
                  type: integer
                passed:
                  type: boolean
                feedback:
                  type: string
      responses:
        '200':
          description: Profile updated
          content:
            application/json:
              schema:
                type: object
```

STEP 5: After pasting, click "Save" or "Create".

SECTION 1.6 — PROMPT REFINEMENT GUIDE (Day 12-17)
────────────────────────────────────────────────────────────────────────────────

After testing, you WILL find failures. Here's how to fix common ones:

PROBLEM: Agent makes up questions instead of calling the tool.
FIX: Add to system prompt: "You MUST call the fetch_question tool for
     EVERY question. You are FORBIDDEN from generating questions from
     your own knowledge. If the tool returns no results, say so."

PROBLEM: Agent gives the answer before the student tries.
FIX: Add: "NEVER reveal the answer, solution steps, or mark scheme
     before the student has submitted their attempt. If the student
     asks for hints, give only the first step's method, not the answer."

PROBLEM: Agent doesn't call update_student_profile after grading.
FIX: Add: "After EVERY grading, you MUST call update_student_profile.
     This is mandatory and silent. Do not ask the student. Just do it."

PROBLEM: Agent gives vague feedback like "Good job!" without detail.
FIX: Add: "Your feedback MUST reference the specific mark scheme point.
     Example: 'You lost the A1 mark because your final answer was 5
     instead of 5.0 as required by the mark scheme.' Never be vague."

PROBLEM: Agent responds to non-academic queries.
FIX: Add: "If the user asks about anything not related to Cambridge
     O/A Level academics (weather, jokes, news, personal questions),
     respond ONLY with: 'I'm Criterion AI, built for Cambridge exam
     prep. Let's get back to studying! Which subject shall we work on?'"

SECTION 2.2 — MONGODB COLLECTION SCHEMAS (BCNF-Normalized)
────────────────────────────────────────────────────────────────────────────────

You have 3 main collections. Below is the exact document structure for each.
These are designed in BCNF: every non-trivial functional dependency has a
superkey as its determinant.

COLLECTION 1: questions
───────────────────────
Purpose: Stores every past paper question with its mark scheme.
Each document = one question (not one paper).

{
    "_id": ObjectId,                      // Auto-generated by MongoDB

    // ── Identification (Candidate Key: syllabus_code + year + session + paper + question_number)
    "syllabus_code": "4024",              // Cambridge syllabus code
    "subject": "Mathematics",             // Human-readable subject name
    "level": "O",                         // "O" or "A"
    "paper": "Paper 1",                   // Paper variant
    "year": 2023,                         // Exam year
    "session": "May/June",                // "May/June", "Oct/Nov", "Feb/Mar"
    "question_number": 7,                 // Question number within the paper

    // ── Content
    "topic": "Quadratic Equations",       // Primary topic
    "subtopic": "Factorisation",          // Optional subtopic for granularity
    "question_text": "Solve 2x² - 5x + 3 = 0 by factorisation.",
    "question_image_url": null,           // URL to image if question has a diagram
    "total_marks": 4,                     // Total marks for this question
    "difficulty": "medium",               // "easy", "medium", "hard"

    // ── Mark Scheme (this is what the grader uses)
    "mark_scheme": {
        "steps": [
            {
                "step_number": 1,
                "description": "Attempt to factorise the quadratic",
                "mark_type": "M",         // M = method, A = accuracy, B = independent
                "marks": 1,
                "acceptable_working": [
                    "(2x - 3)(x - 1)",
                    "2(x - 3/2)(x - 1)"
                ],
                "keywords": ["factorise", "factor", "brackets"],
                "common_errors": ["(2x + 3)(x + 1)", "wrong signs"]
            },
            {
                "step_number": 2,
                "description": "Set each factor equal to zero",
                "mark_type": "M",
                "marks": 1,
                "acceptable_working": [
                    "2x - 3 = 0 or x - 1 = 0",
                    "2x = 3, x = 1"
                ],
                "keywords": ["= 0", "equals zero"],
                "depends_on": [1]         // This M mark depends on step 1 being correct
            },
            {
                "step_number": 3,
                "description": "Solve for x = 3/2",
                "mark_type": "A",
                "marks": 1,
                "acceptable_answers": ["3/2", "1.5"],
                "depends_on": [1, 2]      // A mark requires correct method
            },
            {
                "step_number": 4,
                "description": "Solve for x = 1",
                "mark_type": "A",
                "marks": 1,
                "acceptable_answers": ["1"],
                "depends_on": [1, 2]
            }
        ],
        "final_answer": "x = 3/2 or x = 1",
        "alternative_methods": [
            "Quadratic formula: x = (5 ± √(25-24))/4 = (5 ± 1)/4"
        ]
    },

    // ── Metadata
    "created_at": "2025-01-15T10:00:00Z",
    "source": "Cambridge 4024/12/M/J/23"  // Human-readable reference
}

INDEXES TO CREATE:
  db.questions.createIndex({ "subject": 1, "level": 1, "topic": 1 })
  db.questions.createIndex({ "syllabus_code": 1, "year": 1, "session": 1 })
  db.questions.createIndex({ "difficulty": 1 })
  db.questions.createIndex({ "topic": 1, "subtopic": 1 })


COLLECTION 2: student_profiles
───────────────────────────────
Purpose: One document per student. Tracks their weakness map and history.

{
    "_id": ObjectId,
    "student_id": "stu_abc123",           // Unique student identifier

    // ── Basic Info
    "name": "Ahmed Khan",
    "email": "ahmed@example.com",         // Optional
    "level": "O",                         // Primary level
    "subjects": ["Mathematics", "Physics"],

    // ── Weakness Map (the adaptive engine reads this)
    // Structure: subject → topic → stats
    "weakness_map": {
        "Mathematics": {
            "Quadratic Equations": {
                "attempts": 8,
                "pass": 5,
                "fail": 3,
                "mastery": 0.625,         // pass / attempts
                "streak": 2,             // Consecutive passes (resets on fail)
                "last_attempt": "2025-06-10T14:30:00Z",
                "avg_score_pct": 0.72     // Average percentage score
            },
            "Differentiation": {
                "attempts": 3,
                "pass": 0,
                "fail": 3,
                "mastery": 0.0,
                "streak": 0,
                "last_attempt": "2025-06-09T16:00:00Z",
                "avg_score_pct": 0.25
            }
        }
    },

    // ── Timestamps
    "created_at": "2025-01-01T00:00:00Z",
    "updated_at": "2025-06-10T14:30:00Z"
}

────────────────────────────────────────────────────────────────────────────────
SECTION 2.4 — THE AUTO-GRADER ENGINE (grader.py)
────────────────────────────────────────────────────────────────────────────────

This is your most important algorithm. Think of it like a competitive
programming problem: given two strings (student's working and the mark
scheme), compute the maximum partial credit.

ALGORITHM OVERVIEW:
───────────────────
INPUT:
  - student_working: a multi-line string of the student's steps
  - mark_scheme: the JSON mark scheme from the question document

OUTPUT:
  - A list of StepFeedback objects
  - Total marks awarded
  - Pass/fail boolean

THE MATCHING ALGORITHM:
───────────────────────
This is a TWO-PHASE approach:

PHASE 1 — STRING NORMALIZATION + FUZZY MATCHING (for simple checks)
  For each step in the mark scheme:
    1. Normalize the student's text:
       - Remove all whitespace
       - Convert to lowercase
       - Standardize math notation: "x^2" = "x²", "sqrt" = "√", etc.
       - Convert fractions: "3/2" = "1.5" (store both forms)
    2. Check if any of the "acceptable_working" or "acceptable_answers"
       strings appear in the normalized student text.
    3. Also check for "keywords" (partial credit indicator).
    4. Scoring per step:
       - Exact match with acceptable_working → full marks for that step
       - Keywords found but answer wrong → method mark only (if M mark)
       - No match at all → 0 marks for that step

PHASE 2 — LLM-ASSISTED GRADING (for complex/ambiguous cases)
  If Phase 1's confidence is low (e.g., fewer than 50% of keywords found
  and no exact match), escalate to Gemini for semantic comparison.
  Send a prompt like:
    "You are a Cambridge examiner. Compare this student working:
     [student_working]
     Against this mark scheme step:
     [mark_scheme_step]
     Award 0 or 1 marks. Explain in one sentence."

DEPENDENCY CHAIN:
  Cambridge mark schemes have dependencies. An A (accuracy) mark usually
  requires the preceding M (method) mark.
  - If step 1 (M1) is wrong, step 3 (A1) that depends on step 1 gets 0
    UNLESS the student used a "follow-through" method (wrote the wrong
    number from step 1 but used it correctly in step 3).
  - This is called "FT" (follow-through) in Cambridge marking.
  - To handle FT: if depends_on step failed, check if the student's
    working is internally consistent (their wrong answer from step 1
    was used correctly in step 3). If yes, award the A mark.

```python
# backend/grader.py
"""
Auto-Grader Engine for Cambridge O/A Level mark scheme evaluation.

Algorithm:
  Phase 1: Normalize + fuzzy match student working against mark scheme
  Phase 2: LLM escalation for ambiguous cases (optional, uses Gemini)

Supports:
  - M (method), A (accuracy), B (independent) marks
  - Dependency chains (A marks depending on M marks)
  - Follow-through (FT) marking for internally consistent errors
  - Partial credit at every step
"""

import re
import math
from typing import Any


# ─── Text Normalization ──────────────────────────────────────────────────

def normalize_math_text(text: str) -> str:
    """
    Normalize a mathematical text string for comparison.
    Strips whitespace, lowercases, standardizes notation.
    """
    if not text:
        return ""

    s = text.lower().strip()

    # Remove all spaces
    s = re.sub(r'\s+', '', s)

    # Standardize common math symbols
    replacements = {
        '×': '*', '÷': '/', '−': '-', '–': '-',
        '²': '^2', '³': '^3', '√': 'sqrt',
        'π': 'pi', '∞': 'inf',
    }
    for old, new in replacements.items():
        s = s.replace(old, new)

    # Remove trailing zeros: "5.0" → "5", "3.50" → "3.5"
    s = re.sub(r'(\d+\.\d*?)0+(?=\D|$)', r'\1', s)
    s = re.sub(r'(\d+)\.(?=\D|$)', r'\1', s)

    return s


def extract_numerical_values(text: str) -> list[float]:
    """Extract all numerical values from text, including fractions."""
    values = []

    # Match fractions like 3/2
    for match in re.finditer(r'(-?\d+)\s*/\s*(\d+)', text):
        num, den = int(match.group(1)), int(match.group(2))
        if den != 0:
            values.append(num / den)

    # Match decimals and integers
    for match in re.finditer(r'-?\d+\.?\d*', text):
        try:
            values.append(float(match.group()))
        except ValueError:
            pass

    return values


def values_match(val1: float, val2: float, tolerance: float = 0.001) -> bool:
    """Check if two numerical values match within tolerance."""
    return abs(val1 - val2) < tolerance


# ─── Step Matching ────────────────────────────────────────────────────────

def match_step(
    student_text: str,
    step_schema: dict[str, Any],
) -> dict[str, Any]:
    """
    Compare student's working against a single mark scheme step.

    Returns:
        {
            "matched": bool,
            "confidence": float (0.0 to 1.0),
            "marks_awarded": int,
            "note": str,
            "match_type": "exact" | "keyword" | "numerical" | "none"
        }
    """
    normalized_student = normalize_math_text(student_text)
    marks_available = step_schema.get("marks", 1)
    mark_type = step_schema.get("mark_type", "B")

    # ── Check 1: Exact match against acceptable working/answers ──
    acceptable = (
        step_schema.get("acceptable_working", [])
        + step_schema.get("acceptable_answers", [])
    )
    for acceptable_text in acceptable:
        normalized_acceptable = normalize_math_text(acceptable_text)
        if normalized_acceptable in normalized_student:
            return {
                "matched": True,
                "confidence": 1.0,
                "marks_awarded": marks_available,
                "note": f"Correct — matches mark scheme",
                "match_type": "exact",
            }

    # ── Check 2: Numerical value match ──
    student_values = extract_numerical_values(student_text)
    acceptable_values = []
    for a in acceptable:
        acceptable_values.extend(extract_numerical_values(a))

    numerical_matches = 0
    for sv in student_values:
        for av in acceptable_values:
            if values_match(sv, av):
                numerical_matches += 1
                break

    if acceptable_values and numerical_matches >= len(acceptable_values):
        return {
            "matched": True,
            "confidence": 0.85,
            "marks_awarded": marks_available,
            "note": "Correct numerical answer found in working",
            "match_type": "numerical",
        }

    # ── Check 3: Keyword partial match (for method marks) ──
    keywords = step_schema.get("keywords", [])
    keyword_hits = 0
    for kw in keywords:
        if normalize_math_text(kw) in normalized_student:
            keyword_hits += 1

    if keywords and keyword_hits >= len(keywords) * 0.5:
        # Student showed the right method but maybe wrong answer
        if mark_type == "M":
            return {
                "matched": True,
                "confidence": 0.6,
                "marks_awarded": marks_available,
                "note": "Method identified — mark awarded for approach",
                "match_type": "keyword",
            }
        else:
            return {
                "matched": False,
                "confidence": 0.4,
                "marks_awarded": 0,
                "note": f"Method seen but final answer incorrect for {mark_type} mark",
                "match_type": "keyword",
            }

    # ── Check 4: Common errors ──
    common_errors = step_schema.get("common_errors", [])
    for err in common_errors:
        if normalize_math_text(err) in normalized_student:
            return {
                "matched": False,
                "confidence": 0.8,
                "marks_awarded": 0,
                "note": f"Common error detected: {err}",
                "match_type": "none",
            }

    # ── No match ──
    return {
        "matched": False,
        "confidence": 0.0,
        "marks_awarded": 0,
        "note": f"Working does not match expected {mark_type} mark criteria",
        "match_type": "none",
    }


# ─── Full Grading Pipeline ───────────────────────────────────────────────

def grade_student_working(
    student_working: str,
    mark_scheme: dict[str, Any],
    total_marks: int,
) -> dict[str, Any]:
    """
    Grade a student's complete working against the full mark scheme.

    Handles:
    - Per-step matching with M/A/B marks
    - Dependency chains (A marks depend on M marks)
    - Follow-through (FT) marking
    - Partial credit aggregation

    Args:
        student_working: The student's full answer as a string.
        mark_scheme: The mark_scheme dict from the question document.
        total_marks: Maximum marks for the question.

    Returns:
        Dict with step_feedback list, total awarded marks, pass/fail, etc.
    """
    steps = mark_scheme.get("steps", [])
    if not steps:
        return {
            "awarded_marks": 0,
            "total_marks": total_marks,
            "step_feedback": [],
            "overall_feedback": "No mark scheme steps defined for this question.",
        }

    # Split student working into lines for per-step analysis
    student_lines = [line.strip() for line in student_working.strip().split('\n') if line.strip()]

    # Combine all student working into one block for matching
    # (students don't always separate steps cleanly)
    full_student_text = ' '.join(student_lines)

    step_results: list[dict[str, Any]] = []
    step_awarded: dict[int, bool] = {}  # step_number -> was mark awarded?

    for step in steps:
        step_num = step["step_number"]
        mark_type = step.get("mark_type", "B")
        marks_available = step.get("marks", 1)

        # ── Check dependency chain ──
        depends_on = step.get("depends_on", [])
        dependency_failed = False
        for dep in depends_on:
            if not step_awarded.get(dep, False):
                dependency_failed = True
                break

        # ── Match this step ──
        result = match_step(full_student_text, step)

        # ── Apply dependency rules ──
        if dependency_failed and result["matched"]:
            if mark_type == "A":
                # A mark with failed dependency:
                # Check for follow-through (FT) — did student use their
                # wrong answer consistently?
                # For now, if confidence is high, award FT mark
                if result["confidence"] >= 0.85:
                    result["note"] = (
                        "Follow-through mark awarded — working is "
                        "internally consistent despite earlier error"
                    )
                    # FT marks are awarded
                else:
                    result["marks_awarded"] = 0
                    result["matched"] = False
                    result["note"] = (
                        f"Cannot award {mark_type} mark — "
                        f"dependent step(s) {depends_on} not achieved"
                    )

        step_awarded[step_num] = result["matched"]

        step_results.append({
            "step": step_num,
            "mark_type": mark_type,
            "content": step.get("description", ""),
            "marks_awarded": result["marks_awarded"],
            "marks_available": marks_available,
            "correct": result["matched"],
            "note": result["note"],
        })

    # ── Aggregate ──
    total_awarded = sum(s["marks_awarded"] for s in step_results)
    percentage = round((total_awarded / total_marks * 100), 1) if total_marks > 0 else 0
    passed = total_awarded >= math.ceil(total_marks * 0.5)

    # ── Generate overall feedback ──
    if percentage == 100:
        overall = "Excellent work! Full marks awarded. Your working is clear and complete."
    elif percentage >= 75:
        overall = "Very good attempt. You demonstrated strong understanding with minor errors."
    elif percentage >= 50:
        overall = "Satisfactory attempt. You showed some correct methods but need to be more careful with accuracy."
    elif percentage >= 25:
        overall = "You showed some understanding but several key steps were incorrect or missing."
    else:
        overall = "This topic needs more practice. Review the mark scheme carefully and try similar questions."

    return {
        "awarded_marks": total_awarded,
        "total_marks": total_marks,
        "percentage": percentage,
        "passed": passed,
        "step_feedback": step_results,
        "overall_feedback": overall,
    }
```

SECTION 2.5 — THE ADAPTIVE WEAKNESS TRACKER (weakness_tracker.py)
────────────────────────────────────────────────────────────────────────────────

ALGORITHM: Weighted Topic Priority Score (WTPS)

PROBLEM STATEMENT:
  Given a student's history across N topics, each with (attempts, pass, fail,
  last_attempt_time, avg_score_pct, streak), compute a priority score for
  each topic. The topic with the HIGHEST priority score is the student's
  weakest area and should be practiced next.

FORMULA:
  For each topic T:

  WTPS(T) = (w1 × failure_rate)
          + (w2 × recency_decay)
          + (w3 × attempt_penalty)
          - (w4 × streak_bonus)

  Where:
    failure_rate   = fail / attempts  (range: 0 to 1, higher = weaker)
    recency_decay  = 1 / (1 + days_since_last_attempt * 0.1)
                     (range: 0 to 1, higher = more recent, should practice
                      topics NOT practiced recently, so we use 1 - recency)
                     Actually: recency_factor = 1 - recency_decay
                     (higher if topic was NOT practiced recently)
    attempt_penalty = 1 / (1 + attempts * 0.2)
                     (higher if fewer attempts, meaning undertested topic)
    streak_bonus   = min(streak / 5, 1.0)
                     (if student got 5+ in a row correct, reduce priority)

  Weights (tunable):
    w1 = 0.45  (failure rate matters most)
    w2 = 0.25  (haven't practiced recently matters second)
    w3 = 0.20  (undertested topics matter third)
    w4 = 0.10  (hot streak reduces priority)

  Final normalization: clamp WTPS to [0, 1].

WALKTHROUGH EXAMPLE:
  Topic: "Differentiation"
    attempts = 3, pass = 0, fail = 3, streak = 0
    last_attempt = 2 days ago, avg_score = 25%

    failure_rate = 3/3 = 1.0
    recency_decay = 1 / (1 + 2*0.1) = 1/1.2 = 0.833
    recency_factor = 1 - 0.833 = 0.167
    attempt_penalty = 1 / (1 + 3*0.2) = 1/1.6 = 0.625
    streak_bonus = min(0/5, 1) = 0.0

    WTPS = (0.45 × 1.0) + (0.25 × 0.167) + (0.20 × 0.625) - (0.10 × 0.0)
         = 0.45 + 0.042 + 0.125 - 0.0
         = 0.617

  Topic: "Integration"
    attempts = 10, pass = 8, fail = 2, streak = 4
    last_attempt = 1 day ago, avg_score = 82%

    failure_rate = 2/10 = 0.2
    recency_decay = 1 / (1 + 1*0.1) = 0.909
    recency_factor = 0.091
    attempt_penalty = 1 / (1 + 10*0.2) = 1/3 = 0.333
    streak_bonus = min(4/5, 1) = 0.8

    WTPS = (0.45 × 0.2) + (0.25 × 0.091) + (0.20 × 0.333) - (0.10 × 0.8)
         = 0.09 + 0.023 + 0.067 - 0.08
         = 0.10

  Result: Differentiation (0.617) >> Integration (0.10)
  → Agent should practice Differentiation next. ✓

# backend/weakness_tracker.py
"""
Adaptive Weakness Tracker for Criterion AI.

Uses the Weighted Topic Priority Score (WTPS) algorithm to determine
which topic a student should practice next based on their historical
performance data stored in MongoDB.
"""

from datetime import datetime, timezone
from typing import Any

from database import get_student_profiles_collection


# ─── Tunable weights ─────────────────────────────────────────────────────
W_FAILURE_RATE = 0.45
W_RECENCY = 0.25
W_ATTEMPT_PENALTY = 0.20
W_STREAK_BONUS = 0.10
RECENCY_DECAY_FACTOR = 0.1
ATTEMPT_DECAY_FACTOR = 0.2
STREAK_CAP = 5


def compute_wtps(topic_stats: dict[str, Any]) -> float:
    """
    Compute the Weighted Topic Priority Score for a single topic.

    Higher score = weaker topic = should be practiced next.

    Args:
        topic_stats: {
            "attempts": int,
            "pass": int,
            "fail": int,
            "mastery": float,
            "streak": int,
            "last_attempt": str (ISO-8601),
            "avg_score_pct": float
        }

    Returns:
        WTPS score between 0.0 and 1.0
    """
    attempts = topic_stats.get("attempts", 0)
    if attempts == 0:
        # Never attempted = high priority (we want to test this topic)
        return 0.85

    fails = topic_stats.get("fail", 0)
    streak = topic_stats.get("streak", 0)

    # ── Factor 1: Failure rate (higher = weaker) ──
    failure_rate = fails / attempts

    # ── Factor 2: Recency (higher = NOT practiced recently) ──
    last_attempt_str = topic_stats.get("last_attempt")
    if last_attempt_str:
        try:
            last_attempt_dt = datetime.fromisoformat(last_attempt_str)
            # Make timezone-aware if needed
            if last_attempt_dt.tzinfo is None:
                last_attempt_dt = last_attempt_dt.replace(tzinfo=timezone.utc)
            days_since = (datetime.now(timezone.utc) - last_attempt_dt).days
        except (ValueError, TypeError):
            days_since = 30  # Default to "long ago"
    else:
        days_since = 30

    recency_decay = 1.0 / (1.0 + days_since * RECENCY_DECAY_FACTOR)
    recency_factor = 1.0 - recency_decay  # Higher if NOT practiced recently

    # ── Factor 3: Attempt penalty (higher if fewer attempts = undertested) ──
    attempt_penalty = 1.0 / (1.0 + attempts * ATTEMPT_DECAY_FACTOR)

    # ── Factor 4: Streak bonus (higher streak = topic mastered recently) ──
    streak_bonus = min(streak / STREAK_CAP, 1.0)

    # ── Weighted sum ──
    wtps = (
        W_FAILURE_RATE * failure_rate
        + W_RECENCY * recency_factor
        + W_ATTEMPT_PENALTY * attempt_penalty
        - W_STREAK_BONUS * streak_bonus
    )

    # Clamp to [0, 1]
    return max(0.0, min(1.0, wtps))


def get_weakest_topics(
    student_id: str,
    subject: str,
    top_n: int = 3,
) -> list[dict[str, Any]]:
    """
    Return the top N weakest topics for a student in a given subject,
    ranked by WTPS (highest priority first).

    Args:
        student_id: The student's unique identifier.
        subject: The subject to analyze, e.g. "Mathematics".
        top_n: How many topics to return.

    Returns:
        List of dicts: [{"topic": str, "wtps": float, "mastery": float, "attempts": int}, ...]
    """
    profiles = get_student_profiles_collection()
    profile = profiles.find_one({"student_id": student_id})

    if profile is None:
        return []

    weakness_map = profile.get("weakness_map", {})
    subject_map = weakness_map.get(subject, {})

    if not subject_map:
        return []

    scored_topics = []
    for topic_name, stats in subject_map.items():
        wtps = compute_wtps(stats)
        scored_topics.append({
            "topic": topic_name,
            "wtps": round(wtps, 4),
            "mastery": stats.get("mastery", 0.0),
            "attempts": stats.get("attempts", 0),
            "avg_score_pct": stats.get("avg_score_pct", 0.0),
            "streak": stats.get("streak", 0),
        })

    # Sort by WTPS descending (highest priority first)
    scored_topics.sort(key=lambda x: x["wtps"], reverse=True)

    return scored_topics[:top_n]


def get_recommended_difficulty(
    student_id: str,
    subject: str,
    topic: str,
) -> str:
    """
    Recommend a difficulty level based on the student's mastery.

    mastery < 0.3  → "easy"   (build confidence)
    mastery 0.3–0.7 → "medium" (challenge appropriately)
    mastery > 0.7  → "hard"   (push to mastery)
    """
    profiles = get_student_profiles_collection()
    profile = profiles.find_one({"student_id": student_id})

    if profile is None:
        return "medium"

    mastery = (
        profile.get("weakness_map", {})
        .get(subject, {})
        .get(topic, {})
        .get("mastery", 0.0)
    )

    if mastery < 0.3:
        return "easy"
    elif mastery < 0.7:
        return "medium"
    else:
        return "hard"

const API_BASE = import.meta.env.VITE_API_URL || 'http://localhost:8000'

export interface Message {

SECTION 3.5 — PROGRESS DASHBOARD COMPONENT
────────────────────────────────────────────────────────────────────────────────

This component shows the student's weakness map visually. Great for demos.

```tsx
// frontend/src/components/ProgressDashboard.tsx
import { useEffect, useState } from 'react'
import { getStudentProfile, type StudentProfile, type TopicStats } from '../services/api'

interface Props {
  studentId: string
  subject: string
}

export default function ProgressDashboard({ studentId, subject }: Props) {
  const [profile, setProfile] = useState<StudentProfile | null>(null)
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    getStudentProfile(studentId)
      .then(setProfile)
      .catch(console.error)
      .finally(() => setLoading(false))
  }, [studentId])

  if (loading) {
    return (
      <div className="flex items-center justify-center p-8">
        <div className="w-8 h-8 border-2 border-primary border-t-transparent rounded-full animate-spin" />
      </div>
    )
  }

  if (!profile) {
    return <p className="text-text-muted p-4">No profile data yet. Start practicing!</p>
  }

  const topics = profile.weakness_map?.[subject] || {}

  return (
    <div className="p-4 space-y-4">
      <h3 className="text-lg font-semibold text-text-primary">
        📊 {subject} — Topic Mastery
      </h3>
      <div className="space-y-3">
        {Object.entries(topics).map(([topic, stats]: [string, TopicStats]) => (
          <div key={topic} className="bg-surface-light rounded-xl p-4 border border-border">
            <div className="flex justify-between items-center mb-2">
              <span className="text-sm font-medium text-text-primary">{topic}</span>
              <span className={`text-xs font-bold px-2 py-0.5 rounded-full ${
                stats.mastery >= 0.7
                  ? 'bg-success/20 text-success'
                  : stats.mastery >= 0.4
                  ? 'bg-warning/20 text-warning'
                  : 'bg-error/20 text-error'
              }`}>
                {Math.round(stats.mastery * 100)}%
              </span>
            </div>
            {/* Progress bar */}
            <div className="w-full h-2 bg-surface rounded-full overflow-hidden">
              <div
                className={`h-full rounded-full transition-all duration-500 ${
                  stats.mastery >= 0.7
                    ? 'bg-success'
                    : stats.mastery >= 0.4
                    ? 'bg-warning'
                    : 'bg-error'
                }`}
                style={{ width: `${Math.round(stats.mastery * 100)}%` }}
              />
            </div>
            <div className="flex justify-between mt-2 text-xs text-text-muted">
              <span>{stats.attempts} attempts</span>
              <span>{stats.pass} pass / {stats.fail} fail</span>
              <span>🔥 streak: {stats.streak}</span>
            </div>
          </div>
        ))}
      </div>
    </div>
  )
}
```
